# TwoGo V1.0.3 - Fitur Interaksi Trip & Peta Lokasi

Versi 1.0.3 menghadirkan fitur interaksi modern pada kartu trip (Like dengan double-tap & Clone dengan press-and-hold) serta perbaikan fungsionalitas visualisasi peta Google Maps.

Berikut adalah dokumentasi lengkap dari fitur-fitur baru dan perubahan yang diimplementasikan pada versi ini:

## 1. Mekanisme Double-Tap/Double-Click untuk Menyukai Trip (Like)
- **Pengalaman Pengguna (UX)**: Terinspirasi oleh Instagram, pengguna kini dapat menyukai (like) sebuah trip dengan melakukan **double-tap** (pada perangkat mobile/sentuh) atau **double-click** (pada desktop) langsung di kartu trip.
- **Efek Visual**: Saat double-tap/double-click dideteksi, sebuah ikon hati merah besar (`❤️`) akan muncul di tengah-tengah kartu dengan animasi transisi membesar (*scale-up*) dan memudar (*fade-out*).
- **Integrasi AJAX**: Aksi ini memicu AJAX POST ke API `/trips/{id}/like` secara latar belakang (background) menggunakan `fetch` API.
- **Pembaruan UI Real-Time**:
  - Jumlah total likes pada kartu (selector `.like-count-text`) akan langsung diperbarui secara dinamis tanpa memuat ulang halaman.
  - Jika aksi dilakukan pada halaman detail trip (`public_show`), tombol like utama (`#like-btn` & `#like-icon`) juga akan otomatis ikut aktif dan berganti warna (merah muda bergaya Neo-Brutalism).
- **Sistem Gamifikasi**:
  - Apabila trip disukai oleh pengguna lain, pencipta asli trip akan menerima push notifikasi aktivitas dan mendapatkan XP tambahan (Gamification Points).

## 2. Fitur Press-and-Hold (Tahan) untuk Menyalin Trip/Wishlist (Clone)
- **Kemudahan Duplikasi (Clone)**: Pengguna dapat menyalin/menyimpan trip publik milik orang lain langsung ke wishlist mereka secara instan dengan menekan dan menahan (**press-and-hold / long-press**) kartu trip selama **800ms**.
- **Deteksi Gerakan (Gesture Detection)**: Menerapkan pointer/mouse listener (`touchstart`, `mousedown`, `touchmove`, `mousemove`, `touchend`, `mouseup`) yang memicu timer penahanan.
- **Umpan Balik Visual Premium**:
  - Saat mulai ditekan, kartu trip akan beranimasi menyusut sedikit (`scale(0.97)`) dengan transisi halus.
  - Sebuah progress bar berwarna biru (`#4361EE`) dengan ketebalan 2px bergaya Neo-Brutalism akan muncul di tepi bawah kartu dan bertambah panjang hingga 100% seiring berjalannya waktu penahanan.
  - Label indikator mengambang bertuliskan `⏱️ Tahan untuk menyimpan...` akan muncul di bagian atas kartu.
  - Jika penahanan dilepas sebelum 800ms atau posisi sentuhan/mouse bergeser lebih dari 15px, seluruh status penahanan, progress bar, dan label indikator akan langsung dibatalkan dan menghilang tanpa mengganggu event navigasi/click standar.
- **Efek Keberhasilan**:
  - Setelah 800ms terlewati, progress bar berubah menjadi hijau (`#00D4AA`), label berubah menjadi `📋 Menyalin...`, dan kartu trip akan melakukan efek animasi melengkung menyusut (`scale(0.93) rotate(-1.5deg)`).
  - Mengirimkan getaran singkat (`vibrate` 50ms) jika didukung oleh perangkat keras seluler.
  - Form POST dikirimkan secara otomatis ke endpoint `/trips/{id}/clone`.

## 3. Penanganan Konflik Navigasi & Klik Tertunda (Delayed Single Click)
- **Masalah**: Pada perangkat seluler, menempatkan listener double-tap pada kartu yang dibungkus tag tautan (`<a>`) akan langsung mengarahkan pengguna ke halaman detail pada sentuhan pertama, sehingga merusak gesture double-tap.
- **Solusi**:
  - Mengubah elemen pembungkus kartu dari tag `<a>` menjadi `<div>` dengan atribut `data-url` untuk menyimpan URL target.
  - Menerapkan **Delayed Single Click (300ms)**: Saat pengguna mengetuk satu kali, sistem menunda navigasi selama 300ms menggunakan `setTimeout` untuk memastikan tidak ada ketukan kedua (double-tap) yang menyusul.
  - Jika ketukan kedua terdeteksi sebelum 300ms, timernya dibatalkan (`clearTimeout`) dan aksi Like dijalankan alih-alih navigasi.
  - Membocorkan elemen klik: Sistem secara cerdas mengabaikan penundaan navigasi jika pengguna secara eksplisit menekan elemen tombol (`<button>`) atau tautan profil pengguna (`<a>`) di dalam kartu.

## 4. Visualisasi & Embed Google Maps Pintar (Smart Location Embed)
- **Peta Interaktif**: Jika kegiatan memiliki **Nama Lokasi** (`location_name`), sistem akan secara otomatis menampilkan pratinjau peta interaktif menggunakan Google Maps Embed iframe.
- **Penanganan URL Pendek**: Google Maps versi pendek (seperti `maps.app.goo.gl`) memicu error X-Frame-Options saat dimuat langsung dalam iframe. Untuk mengatasinya:
  - Jika **Nama Lokasi** diisi, pratinjau iframe di-render menggunakan query pencarian alamat/koordinat (`https://maps.google.com/maps?q=...&output=embed`), yang sepenuhnya aman dan didukung oleh Google Maps.
  - Area pratinjau peta ditutup overlay transparan agar seluruh kartu peta responsif saat ditekan untuk navigasi di tab baru.
- **Fallback Tombol**: Jika pengguna hanya menginputkan **Link Google Maps** tanpa nama lokasi, sistem akan menyembunyikan pratinjau peta (karena tidak dapat dimuat di iframe) dan menampilkannya sebagai tombol aksi Neo-Brutalism "🗺️ Buka di Google Maps".
- **Panduan Pengguna**: Menambahkan kotak pesan edukasi dengan tautan edit bagi pembuat trip agar mereka tahu bahwa mengisi nama lokasi akan mengaktifkan pratinjau peta otomatis.
