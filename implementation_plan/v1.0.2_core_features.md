# TwoGo V1.0.2 - Core Features Documentation

Versi 1.0.2 membawa beberapa peningkatan signifikan terutama pada pengalaman pengguna, manajemen dokumentasi, dan perbaikan tampilan. Berikut adalah ringkasan fitur inti yang ditambahkan atau diubah pada versi ini:

## 1. Mekanisme Wishlist Berbasis Tanggal (Date-based Wishlist)
- **Konsep Baru**: Sebuah trip dianggap sebagai "Wishlist" jika **Tanggal Berangkat (Start Date)** dan **Tanggal Pulang (End Date)** dikosongkan saat pembuatan.
- Halaman form pembuatan trip disesuaikan: Tanggal bersifat opsional.
- Jika trip berada dalam status Wishlist, pengguna dapat menambahkan tanggal kemudian melalui tombol "Isi Tanggal" yang akan langsung memindahkan statusnya menjadi trip aktif.
- Tab Wishlist telah dipindahkan ke halaman utama "Trip Kamu" (`trips.index`) agar lebih terpusat dan mudah diakses.

## 2. Peningkatan Fitur Dokumentasi & Pencatatan Budget Real
- **UI Tab Summary**: Halaman Ringkasan Trip kini memiliki dua tab: `Ringkasan` dan `Dokumentasi`.
- **Kompresi Pintar (Smart Compression)**: 
  - Batas ukuran upload foto diperbesar menjadi **20MB** untuk mendukung gambar dari smartphone resolusi tinggi.
  - Gambar asli di-resize (maksimal lebar 1920px) secara otomatis agar *storage* server tetap optimal.
  - Di-generate *thumbnail preview* kecil (maksimal lebar 400px) khusus untuk tampilan *grid* di tab dokumentasi.
- **Photo Grid & Modal**: Foto-foto kegiatan yang selesai akan masuk ke galeri dokumentasi dengan UI Grid. Saat diklik, akan muncul pop-up preview resolusi tinggi beserta tombol untuk mengunduhnya.

## 3. Global Error Handling (Validation Toast)
- Pesan kesalahan validasi (seperti gagal upload file yang tidak sesuai format atau terlalu besar melebihi 20MB) kini ditampilkan dengan pop-up Toast berwarna merah di sudut kanan atas layar.
- Perubahan ini membuat notifikasi error lebih menonjol dibandingkan pesan teks standar di bawah input.

## 4. UI/UX Refinement (Custom Modals & Neo-Brutalism consistency)
- Menu "Budget" di Bottom Navbar diganti menjadi "Search".
- Browser `confirm()` bawaan untuk **Menghapus Trip**, **Menghapus Kegiatan**, dan **Membatalkan (Uncheck) Kegiatan** sepenuhnya digantikan dengan *custom pop-up modal* bergaya Neo-Brutalism (border tebal hitam, *shadow box*, dll).
- Membatalkan (uncheck) kegiatan yang sudah diselesaikan akan otomatis **menghapus foto** dan mereset **budget real** terkait.
